# ربات فروش فیلترشکن (V2Ray) برای تلگرام

این ربات با **Python + aiogram 3** نوشته شده و این قابلیت‌ها رو داره:

- 🛒 **خرید سرویس**: کاربر پلن رو انتخاب می‌کنه، ربات خودکار روی پنل **3X-UI** کلاینت VLESS+Reality می‌سازه و لینک/QR Code رو می‌فرسته.
- 💰 **کیف پول**: کاربر مبلغ دلخواه رو کارت‌به‌کارت واریز می‌کنه، عکس رسید رو می‌فرسته، ادمین تایید/رد می‌کنه، موجودی به‌صورت خودکار شارژ میشه.
- 🎫 **پشتیبانی**: پیام کاربر به‌صورت تیکت برای ادمین ارسال میشه و ادمین با یک دستور پاسخ می‌ده.
- ⚙️ **پنل مدیریت** (فقط ادمین): افزودن/حذف محصول، دیدن درخواست‌های شارژ، دیدن و پاسخ به تیکت‌ها.

دیتابیس **SQLite** هست (فایل `bot_database.db`) و نیازی به نصب جداگانه نداره.

---

## ۱. پیش‌نیازها روی سرور

سروری که قبلاً از دیاکو وب خریدی و پنل **3X-UI** روش نصب کردی کافیه. اگر پایتون نداره:

```bash
apt update && apt install python3 python3-pip python3-venv -y
```

## ۲. آپلود پروژه روی سرور

کل پوشه `v2ray_shop_bot` رو روی سرور آپلود کن (با `scp` یا FileZilla)، یا با `git`/`unzip` اگه زیپ فرستادی:

```bash
unzip v2ray_shop_bot.zip
cd v2ray_shop_bot
```

## ۳. نصب پکیج‌ها

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## ۴. ساخت ربات تلگرام

1. به [@BotFather](https://t.me/BotFather) پیام بده → `/newbot` → یک اسم و یوزرنیم انتخاب کن.
2. توکنی که می‌ده رو کپی کن.

## ۵. گرفتن آیدی عددی خودت (برای ادمین)

به [@userinfobot](https://t.me/userinfobot) پیام بده، آیدی عددیت رو می‌ده.

## ۶. تنظیم فایل `config.py`

فایل `config.py` رو باز کن و این مقادیر رو پر کن:

```python
BOT_TOKEN = "توکن ربات از BotFather"
ADMIN_IDS = [آیدی عددی خودت]
CARD_NUMBER = "شماره کارتت برای دریافت شارژ"
CARD_HOLDER = "نام صاحب کارت"
PANEL_URL = "http://آی‌پی‌سرور:پورت‌پنل"
PANEL_USERNAME = "یوزرنیم پنل 3X-UI"
PANEL_PASSWORD = "پسورد پنل 3X-UI"
```

## ۷. ساخت Inbound در پنل 3X-UI (یک‌بار، قبل از افزودن محصول)

داخل پنل 3X-UI یک Inbound با پروتکل **VLESS** و امنیت **Reality** بساز (همونی که قبلاً راهنماییت کردم).
آیدی همین Inbound رو لازم داری تا موقع تعریف محصول در ربات وارد کنی (در خود پنل، کنار اسم Inbound، یک عدد به‌عنوان ID نشون داده میشه).

## ۸. اجرای ربات

```bash
python3 main.py
```

اگه پیام خطا نگرفتی و در تلگرام به ربات `/start` زدی و منو اومد، یعنی موفق بودی 🎉

## ۹. روشن نگه‌داشتن ربات (بعد از قطع شدن SSH)

برای اینکه با بستن ترمینال ربات خاموش نشه، با `screen` اجرا کن:

```bash
apt install screen -y
screen -S bot
source venv/bin/activate
python3 main.py
```
بعد `Ctrl+A` و `D` بزن تا از screen خارج بشی بدون اینکه ربات قطع بشه.
برای برگشتن به محیط: `screen -r bot`

(روش حرفه‌ای‌تر و پایدارتر، ساخت یک سرویس systemd هست — اگه خواستی برات می‌سازم.)

## ۱۰. افزودن اولین محصول

در تلگرام به ربات برو، چون آیدیت داخل `ADMIN_IDS` هست دکمه «⚙️ پنل مدیریت» رو می‌بینی → «➕ افزودن محصول» → مراحل رو طی کن (اسم، قیمت، مدت، ترافیک، آیدی Inbound، توضیحات).

## دیپلوی روی Railway (دائمی، بدون نیاز به نگه‌داشتن سرور روشن با screen)

### مرحله ۰: مهم — دسترسی ریلوی به پنل سرورت
ربات روی سرورهای ریلوی اجرا میشه، نه روی همون VPS که پنل 3X-UI روشه. پس باید:
- پورت پنل 3X-UI روی فایروال VPS، **به همه باز** باشه (نه فقط IP خودت)، چون آی‌پی ریلوی ثابت نیست.
- امنیتش رو با پسورد قوی پنل تامین کن؛ بهتره SSL (HTTPS) هم روی پنل فعال کنی.
```bash
ufw allow PORT_PANEL
```

### مرحله ۱: آپلود پروژه به گیت‌هاب
1. یک ریپازیتوری جدید (می‌تونه Private باشه) بساز.
2. محتوای پوشه `v2ray_shop_bot` رو داخلش push کن (فایل `config.py` با مقادیر پیش‌فرض مشکلی نداره چون مقادیر واقعی از Variables ریلوی خونده میشه، نه از این فایل).

### مرحله ۲: ساخت پروژه در Railway
1. وارد [railway.app](https://railway.app) شو → **New Project** → **Deploy from GitHub repo** → ریپازیتوری رو انتخاب کن.
2. ریلوی به‌خاطر وجود `requirements.txt` و `Procfile`، خودش محیط پایتون رو می‌سازه.

### مرحله ۳: تنظیم Variables
داخل پروژه‌ی ریلوی → تب **Variables** → این مقادیر رو اضافه کن (دقیقاً مثل `.env.example`):

| متغیر | مقدار |
|---|---|
| `BOT_TOKEN` | توکن از BotFather |
| `ADMIN_IDS` | آیدی عددی خودت (اگه چندتا: `111,222`) |
| `CARD_NUMBER` | شماره کارتت |
| `CARD_HOLDER` | نام صاحب کارت |
| `PANEL_URL` | آدرس پنل 3X-UI، مثل `http://IP:PORT` |
| `PANEL_USERNAME` | یوزرنیم پنل |
| `PANEL_PASSWORD` | پسورد پنل |
| `DB_PATH` | `/data/bot_database.db` |

### مرحله ۴: افزودن Volume (برای حفظ دیتابیس)
بدون این مرحله، با هر دیپلوی جدید کل کیف‌پول‌ها و سفارش‌ها پاک میشه.
1. در پروژه ریلوی → تب **Settings** سرویس → بخش **Volumes** → **New Volume**.
2. Mount Path رو بذار: `/data`
3. دیپلوی رو دوباره بزن (Redeploy).

### مرحله ۵: اجرا
ریلوی به‌خاطر `Procfile` (`worker: python3 main.py`) ربات رو به‌صورت یک **Worker** (نه Web Service) اجرا می‌کنه — یعنی نیازی به پورت باز کردن نداره، فقط دائم در حال polling از تلگرامه.
وضعیت اجرا رو از تب **Deployments → Logs** می‌تونی ببینی؛ اگه نوشت ربات بالا اومده و خطایی نبود، یعنی تمومه.

### جایگزین: دیپلوی مستقیم بدون گیت‌هاب (با Railway CLI)
```bash
npm install -g @railway/cli
railway login
cd v2ray_shop_bot
railway init
railway up
```
بعدش همون مرحله‌ی Variables و Volume رو از داشبورد انجام بده.

## نکات مهم امنیتی و عملیاتی

- فایل `config.py` شامل اطلاعات حساسه (پسورد پنل، توکن ربات) — جایی عمومی (مثل گیت‌هاب پابلیک) آپلودش نکن.
- قبل از فروش واقعی، حتماً خودت یک بار کل مسیر خرید رو تست کن (با موجودی تستی که دستی به کیف پولت اضافه می‌کنی).
- برای افزایش امنیت پنل 3X-UI، پورت پنل رو غیرپیش‌فرض بذار و فقط از IP خودت بهش دسترسی بده (با ufw).
- این نسخه یک پایه‌ی کاربردی و قابل توسعه‌ست؛ برای حجم فروش بالا بهتره از PostgreSQL به‌جای SQLite و از وب‌هوک به‌جای polling استفاده کنی.

## افزودن دستی موجودی به کیف پول (برای تست)

اگه می‌خوای بدون فرآیند رسید، به خودت موجودی تستی بدی، با Python:

```bash
python3 -c "import database as db; db.init_db(); db.change_balance(YOUR_TELEGRAM_ID, 100000)"
```
